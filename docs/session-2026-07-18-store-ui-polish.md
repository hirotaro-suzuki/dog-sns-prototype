# 2026-07-18 店舗側UI総点検と改善（誤操作防止・案内表示・軽量化）

最終更新: 2026-07-18

店舗側UI（ログイン → 撮影 → 編集 → 完成・保存）をコード全体で総点検し、
「忙しい店員が迷わない・誤操作しない・サクサク動く」の基準で10件を指摘、
同日中に10件すべてを実装した記録。

## 点検で挙げた10件と判断

| # | 内容 | 判断 |
|---|---|---|
| 1 | ゴミ箱（やり直し）ボタンが確認なしで撮影写真を全部消す | 対応済み（確認ダイアログ追加） |
| 2 | ログアウトが「×」アイコンで確認なし | 対応済み（退出アイコン化＋確認ダイアログ） |
| 3 | 担当者未選択だとシャッターが無反応で理由の案内がない | 対応済み（カメラ上に案内表示） |
| 4 | 完成画面にも枠選択バーが出て、押すと保存データの枠URLだけ変わる | 対応済み（その後第5段で編集画面からも撤去し、枠選択は撮影画面のみに） |
| 5 | モザイク描画が点ごとに使い捨てキャンバスを生成し重い | 対応済み（作業用キャンバス1枚を使い回し） |
| 6 | 文字編集パネルが小さな「×」でしか閉じられない | 対応済み（枠外タップで選択解除） |
| 7 | 未ログイン時に「店舗ログインへ」ボタン画面を1枚挟む | 対応済み（ログイン画面へ直接リダイレクト） |
| 8 | 写真3枚で画面が暗くなるだけで次の行動案内がない | 対応済み（「下の写真をタップして1枚選んでください」表示） |
| 9 | アイコンの意味の重複（×・ゴミ箱の多役） | 対応済み（下記の意味分け表に統一） |
| 10 | 「写真位置リセット」アイコンが全画面表示と紛らわしい | 対応済み（中心合わせの照準アイコンへ変更） |

## 実装の内訳（3コミット）

1. `ba56b83` 第1段: 事故・迷い防止（#1〜#4）
   - やり直し（ゴミ箱）とログアウトに `window.confirm` の確認を追加。
     確認は標準ダイアログを採用（部品を増やさない。凝った確認画面は必要になったら差し替え）。
   - ログアウトアイコンを「×」から一般的な退出（ドアと矢印）アイコンへ変更（3箇所）。
   - 担当者未選択の間、カメラ映像上に「担当者を選ぶと撮影できます」を表示（`.camera-stage-hint` 追加）。
   - 枠選択バーを `MosaicCanvas` の `frameChooser` プロパティ経由に変更し、編集画面でだけ描画。
     完成画像（焼き付け済み）と保存される枠URLが食い違う穴を塞いだ。
2. `958718f` 第2段: モザイク描画の軽量化（#5）
   - `drawMosaicSpot` が再描画のたびに点ごとに `canvas` 要素を生成していたのを、
     モジュール内で1枚だけ遅延生成して使い回す形へ。描画結果は不変。
3. `1718a5a` 第3段: 快適さ（#6〜#8）
   - キャンバスの文字以外の場所をタップしたら文字選択を解除してパネルを閉じる。
   - `/store` 未ログイン時は案内画面を出さず `window.location.replace("/store/login")` で直行
     （replaceなので戻るボタンでリダイレクトへ戻らない）。
   - 写真3枚保持時の暗転オーバーレイに案内文を追加。

4. 第4段（同日追加）: アイコンの意味統一（#9・#10）
   - 意味分けを次のとおり統一した:
     - **ゴミ箱** = 1個消す（写真プレビューの削除、文字の削除、サムネイル右上の削除バッジ）
     - **×** = 閉じる（文字編集パネルのみ）
     - **丸い矢印（反時計回り）** = 撮影からやり直す（旧「キャンセル」のゴミ箱を置き換え。編集・完成の両画面）
     - **ドアと矢印** = ログアウト（第1段で対応済み）
   - 「写真位置をリセット」は四隅アイコン（全画面表示と紛らわしい）から、
     中心合わせの照準アイコン（円＋十字の目盛り）へ変更。
5. 第5段（同日追加、ユーザー指摘）: 編集画面をスクロール無しに
   - 編集画面で縦スクロールが必要になり分かりにくい、という実機フィードバックを受けて対応。
   - 編集画面の枠選択バーを廃止し、枠選択は撮影画面のみとした（ユーザー提案）。
     編集中に枠を変えたい場合は「戻る」で撮影画面へ戻って選び直す（稀なケースと判断）。
   - あわせてキャンバスを「残りの高さに収まる正方形」に変更（撮影画面のcamera-stageと同方式:
     `.canvas-frame-wrap` flex + `aspect-ratio: 1/1`）。枠バー削除だけでは画面サイズ次第で
     はみ出しが残り得るため、レイアウト側でスクロール無しを保証した。

対象ファイル: `src/components/CameraCapture.tsx`、`src/components/MosaicCanvas.tsx`、
`src/components/StoreHome.tsx`、`src/app/globals.css`
