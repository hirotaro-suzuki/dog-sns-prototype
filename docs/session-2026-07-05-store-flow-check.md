# 2026-07-05 店舗側iPadフロー最終チェック

最終更新: 2026-07-05

この文書は、店舗側 `/store` の写真撮影から保存までの流れを本番前に見直した作業ログである。

GitHub、Vercel、Supabaseを正とする。ローカルPC、Dropbox、手元フォルダを正本にしない。

## 目的

飲食店の現場スタッフが、iPadで迷わず以下を行えるようにする。

1. 店舗ログイン
2. 写真撮影
3. 写真選択
4. 担当者選択
5. 枠確認と画像編集
6. 完成画像確認
7. 必要に応じて印刷
8. お客様からSNS掲載OKをもらう
9. クラウド保存
10. 次の撮影または終了

## 反映したこと

### 店舗ホーム

本番店舗スタッフ向けに、技術情報表示を外した。

外した表示:

- 店舗ID
- ロゴURL
- フレームURL
- テーマ色などのDB確認用情報

店舗ホームでは、店舗名、担当者数、撮影開始、ログアウトを中心にした。

### 撮影

3枚撮り切らなくても、1枚または2枚の時点で写真選択へ進めるようにした。

追加した操作:

- `この写真で選ぶ`

これにより、忙しい現場でよい写真が撮れた時点で次へ進める。

### 担当者選択

担当者が0名の場合の案内を強めた。

表示方針:

- 担当者が登録されていない場合は、本部画面で担当者登録が必要だと分かるようにする。
- 担当者未選択では画像編集へ進めない。

### 枠

撮影前だけでなく、画像編集前の段階でも枠を選び直せるようにした。

枠を変えると、編集画面の完成画像にも選択中の枠が反映される。

### 完成画像確認と保存

完成画像をiPad画面上でお客様に確認してもらえるため、保存前に必ず印刷しないと進めない条件は外した。

現在の流れ:

1. 完成画像を作る
2. お客様に完成画像を画面または印刷で確認してもらう
3. `お客様からSNS掲載OKをもらいました` にチェックする
4. `SNS掲載OKを確認したので保存` でクラウド保存する

印刷は任意操作とする。印刷した場合は `printed_at` が入る。印刷しない場合は `printed_at` は `null` で保存される。

保存ボタンは、SNS掲載OKチェック後でないと押せない。印刷の有無では保存を止めない。

### 保存後

保存後に次の操作へ迷わないようにした。

追加した操作:

- `次のわんちゃんを撮る`
- `撮影を終了する`

保存後は、誤って編集や写真選択へ戻らないように、編集戻りや写真選択戻りを無効化する。

## 2026-07-05 追加修正

### 枠削除

管理画面の枠タブに、停止中の枠だけ削除できる操作を追加した。

方針:

- 有効な枠は削除できない。
- 削除したい場合は、先に有効を外して保存する。
- 削除対象は `store_frames` のDB行で、アップロード済み画像ファイルのStorage削除は別作業とする。

これにより、テスト用・初期作成用の停止中枠が増えても、管理画面上から整理できる。

ユーザー確認で、停止中の枠だけ削除でき、有効中の枠は削除できないことを確認済み。

### iPad編集画面の紫帯

実際の枠画像が読み込めている場合は、システム側の仮デザイン帯を重ねないようにした。

変更後:

- 枠画像がある場合は、枠画像を優先して描画する。
- 日付表示と担当者表示は残す。
- 枠画像が読めない場合は、従来のテーマ色ベースの仮フレームを使う。

ユーザー確認で、実枠画像を選んだ時に上下の紫帯が表示されないことを確認済み。

### 文字水平化の再描画

写真を回転した後、選択中の文字だけを `水平` ボタンで画面上水平に戻す機能がiPad上で効いていないと報告された。

原因候補:

- Canvas上の文字は写真の回転角度と文字自身の角度を合成して描画している。
- 選択中の文字入力欄はCanvasとは別のHTMLレイヤーで表示している。
- 写真の移動、拡大縮小、回転は `transformRef` に保持しており、写真回転時にReactの再描画が必ず走る状態ではなかった。

対応:

- 写真の移動、拡大縮小、回転後に、選択中の文字編集UIも再計算されるようにした。
- 写真位置リセット時にも、選択中の文字編集UIを再計算するようにした。

確認状況:

- GitHub main反映済み。
- 対象コミット `bf0d5ef6b9fd5dec40343481d9df22c85f49cad6` のVercel statusは `success`。
- ユーザー確認で、iPad Safari上の水平ボタンが期待通り動くことを確認済み。

### 保存前の印刷必須条件を解除

お客様が完成画像をiPad画面上で確認できるため、保存前に必ず印刷しないと進めない条件は現場運用に合わないと判断した。

変更後:

- 印刷は任意操作。
- 完成画像を画面または印刷で確認してもらう。
- `お客様からSNS掲載OKをもらいました` のチェック後に保存できる。
- 印刷していない場合、`assets.printed_at` は `null` で保存される。

確認状況:

- GitHub main反映済み。
- 対象コミット `20d0e00` のVercel statusは `success`。
- iPad Safariで、印刷せずに保存まで進めることはユーザー実機確認待ち。
