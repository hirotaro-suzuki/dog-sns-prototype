# 2026-07-05 店舗側iPadフロー最終チェック

最終更新: 2026-07-05

この文書は、店舗側 `/store` の写真撮影から保存までの流れを本番前に見直した作業ログである。

GitHub、Vercel、Supabaseを正とする。ローカルPC、Dropbox、手元フォルダを正本にしない。

## 目的

飲食店の現場スタッフが、iPadで迷わず以下を行えるようにする。

1. 店舗ログイン
2. 写真撮影
3. 写真選択
4. 担当者選択
5. 枠確認と画像編集
6. 完成画像確認
7. 印刷
8. お客様からSNS掲載OKをもらう
9. クラウド保存
10. 次の撮影または終了

## 反映したこと

### 店舗ホーム

本番店舗スタッフ向けに、技術情報表示を外した。

外した表示:

- 店舗ID
- ロゴURL
- フレームURL
- テーマ色などのDB確認用情報

店舗ホームでは、店舗名、担当者数、撮影開始、ログアウトを中心にした。

### 撮影

3枚撮り切らなくても、1枚または2枚の時点で写真選択へ進めるようにした。

追加した操作:

- `この写真で選ぶ`

これにより、忙しい現場でよい写真が撮れた時点で次へ進める。

### 担当者選択

担当者が0名の場合の案内を強めた。

表示方針:

- 担当者が登録されていない場合は、本部画面で担当者登録が必要だと分かるようにする。
- 担当者未選択では画像編集へ進めない。

### 枠

撮影前だけでなく、画像編集前の段階でも枠を選び直せるようにした。

枠を変えると、編集画面の完成画像にも選択中の枠が反映される。

### 完成画像確認

保存より先に印刷を行う操作順へ変更した。

現在の流れ:

1. 完成画像を作る
2. 印刷する
3. お客様に完成画像を確認してもらう
4. `お客様からSNS掲載OKをもらいました` にチェックする
5. `SNS掲載OKをもらったので保存` でクラウド保存する

保存ボタンは、印刷後かつSNS掲載OKチェック後でないと押せない。

### 保存後

保存後に次の操作へ迷わないようにした。

追加した操作:

- `次のわんちゃんを撮る`
- `撮影を終了する`

保存後は、誤って編集や写真選択へ戻らないように、編集戻りや写真選択戻りを無効化する。

## 2026-07-05 追加修正

### 枠削除

管理画面の枠タブに、停止中の枠だけ削除できる操作を追加した。

方針:

- 有効な枠は削除できない。
- 削除したい場合は、先に有効を外して保存する。
- 削除対象は `store_frames` のDB行で、アップロード済み画像ファイルのStorage削除は別作業とする。

これにより、テスト用・初期作成用の停止中枠が増えても、管理画面上から整理できる。

### iPad編集画面の紫帯

実際の枠画像が読み込めている場合は、システム側の仮デザイン帯を重ねないようにした。

変更後:

- 枠画像がある場合は、枠画像を優先して描画する。
- 日付表示と担当者表示は残す。
- 枠画像が読めない場合は、従来のテーマ色ベースの仮フレームを使う。
