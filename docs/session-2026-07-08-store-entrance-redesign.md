# 2026-07-08 店舗側「入口」画面デザイン刷新

最終更新: 2026-07-08

この文書は、店舗側iPad画面のうち「ログイン→店舗ホーム→撮影」までの見た目を刷新した作業ログである。
前回セッション（`docs/session-2026-07-06-square-canvas.md`）はAdmin機能と枠設定が中心だったが、今回はデザイン・UI面のブラッシュアップが目的。

GitHub、Vercel、Supabaseを正とする。ローカルPC、Dropbox、手元フォルダを正本にしない。

## 背景

Codexでベースを作った経緯があり、機能優先でデザインがほぼ未着手だった。ユーザーから「入口の印象優先」で全体の配色・世界観を作り直したいという要望があった。

## 決定事項（実装前に合意した内容）

### 全体方針

- 対象範囲は今回「入口」＝ログイン画面・店舗ホーム・撮影画面。編集画面(MosaicCanvas)やAdmin画面のレイアウトは今回は変更しない（共通クラス経由で配色の恩恵は受ける）。
- 配色・世界観は全面刷新してよい（参考ブランドなし、Claude Codeに一任）。
- 全画面共通で「スクロールが発生しない」レイアウトにする。`100dvh`基準のflexレイアウトを基本とし、カメラ枠(正方形)はビューポート高さから逆算してサイズを決める。
- 店舗ごとのテーマ色上書き（`store.themeColor` → CSS変数 `--accent` / `--accent-dark`）の仕組みは維持する。

### 配色案

- 背景: 温かみのあるクリーム `#FAF6F0`
- パネル: 白、柔らかい暖色シャドウ
- 文字: 暖かいチャコール `#2B2420` / 補助テキスト `#8A7F72`
- メインアクセント: テラコッタ `#E8734A`（ホバー・強調時は `#C85A34`）
- 枠線: 薄いベージュ `#EDE3D6`
- 角丸を全体的に大きく（8px→14〜16px程度）
- 見出しに丸みのある和文フォント（Zen Maru Gothic系）を採用

### ログイン画面

- 店舗コードは`localStorage`に前回ログイン成功時の値を保存し、次回自動プリセット。フィールド下に「前回: ○○」の小さい表示を添えて、誤った店舗コードのままログインする事故を防ぐ。
- PINは記憶しない（毎回入力）。
- PIN入力欄は数字以外を`onChange`で除去するフィルタを入れる。表示/非表示切り替え（目のアイコン）を付ける。
- 入力欄はカード型・角丸・インナーシャドウ・フォーカス時にアクセントカラーで光るリング、という「iPadアプリのフォーム」らしい見た目にする。

### 撮影画面

- 常時表示の説明文（notice）は撤去。エラー時のみ最小限表示する。
- 店舗名表示: `store.logoUrl`があればロゴ＋店名の小バッジ表示にする案があったが、現状ロゴデータを正式に持つ店舗が無いため**当面は店舗名テキストのみ表示**とする（ロゴ対応は将来Admin側の整備が必要になった時点で再検討）。
- シャッターボタンはカメラアプリ風の大きな丸ボタン。
  - 押下後、blobエンコード完了までの一瞬だけ無効化して連打を防止する（タップ自体の視覚フィードバックは残す）。
- サムネイルはシャッターのすぐ近くに撮影直後から表示する（エンコード完了次第、即追加）。
- **サムネイルタップ→全画面プレビュー**（黒背景に写真をcontain表示）という共通コンポーネントを新設する。
  - 撮影中のサムネイルをタップした場合: 「これに決める」（そのままスタッフ選択画面へ直行）／「この写真を消す」（削除して撮影画面に戻る）
  - この全画面プレビューが「1枚だけ選ぶ」動線を兼ねるため、**専用の写真選択画面（`PhotoPicker.tsx`、"pick"ステップ）は丸ごと廃止**する。
- 以下のボタンは廃止する。
  - 「この写真で選ぶ」（サムネイルタップ→全画面プレビューの「これに決める」に統合されるため不要）
  - 「やり直し」（当初「直前の1枚を取り消す」機能だと誤解されていた。実装は全破棄+カメラ再起動で、意図とズレていたため廃止。全画面プレビューの「この写真を消す」が実質の代替になる）
  - 「キャンセル」（`retakePhotos`と全く同じ処理だったため廃止）
  - 「店舗ホームに戻る」（店舗ホーム自体が中継地点で紛らわしいため撮影画面からは撤去。ログアウトのみ残す）
- 3枚撮り切ってまだ何も決めていない状態は、シャッターをグレーアウト＋サムネイル枠のハイライトなど**視覚のみ**で伝える。説明文は出さない（スタッフは固定メンバーで1日10枚程度撮る想定のため、すぐ慣れる）。

### 追加決定事項（実機確認後、2026-07-08）

最初の実装をユーザーが実機（iPad Safari）で確認したフィードバックを受けて、追加で合意した内容。`docs/project-principles.md`「5. 現場操作を軽くする」にも方針として反映済み。

- 担当者選択を撮影前（撮影画面内）に移す。シャッターは担当者を選ぶまで押せない。これにより、撮影後の「担当者を選ぶ」専用画面（`DogInfoForm.tsx`）を廃止し、サムネイル全画面プレビューの「これに決める」から画像編集へ直行できるようにする。
- ボタン等は可能な限り文言ではなく記号・アイコンで表す（例: ログアウトは✕）。ただし担当者名やエラーメッセージなど文章でしか伝わらない情報までは記号化しない。
- 店舗はレストラン店内でお客様の目にも触れるため、見た目のプロっぽさを重視する。
- 各店舗1日10回程度・担当者はほぼ固定という利用実態を前提に、説明文が無くても繰り返し使えばすぐ覚えられるUIを優先する。

### 実機確認で見つかった不具合（修正済み）

- 撮影した写真の保存データが、カメラ枠内の見た目（正方形にcoverで切り抜き表示）と一致しておらず、全画面プレビューで違う形の画像に見えていた。撮影時に中央から正方形へ切り抜いて保存するよう修正。
- 撮影画面のカメラ枠サイズを`min(100%, 62dvh)`という固定の当て推量値で決めていたため、実機ではヘッダー等と合計して画面高を超え、スクロールが発生していた。flexboxで残り領域に自動フィットする方式に変更。

## 実装状況

2026-07-08、決定事項どおりに実装し、ローカルビルド（`npm run build`）成功まで確認済み。

- `src/app/globals.css`: 配色（クリーム背景／テラコッタアクセント）、角丸拡大（`--radius`変数化）、見出しフォント変数、ログイン入力欄のインナーシャドウ＋フォーカスリング、店舗側画面用no-scrollレイアウト（`.store-app-shell`）、丸型シャッター・サムネイル帯・全画面プレビューのスタイルを追加。廃止した`PhotoPicker`専用の`.picker-panel`/`.photo-choice`等は削除。
- `src/app/layout.tsx`: Zen Maru Gothicフォントを`<link>`で読み込み（`next/font/google`はCJKグリフの収録範囲が不安定なため、実行時にGoogle Fonts CDNから読む方式にした）。
- `src/components/StoreLoginForm.tsx` / `src/app/store/login/page.tsx`: 店舗コードをlocalStorage（`dog-sns-last-login-code`）に記憶し次回自動プリセット＋「前回: ○○」表示。PINは数字以外を`onChange`で除去し、目のアイコンで表示/非表示切替。
- `src/components/PhotoPreviewOverlay.tsx`（新設）: 黒背景に写真をcontain表示する全画面プレビュー。「これに決める」でスタッフ選択へ、「この写真を消す」で削除、×で閉じる。
- `src/components/CameraCapture.tsx`: `pick`ステップを廃止し、サムネイルタップ→`PhotoPreviewOverlay`という動線に統合。「この写真で選ぶ」「やり直し」「キャンセル」「店舗ホームに戻る」の各ボタンを撮影画面から削除（ログアウトのみ残す）。シャッターは丸型ボタン化し、`isEncodingRef`によるガードで連打時の多重撮影のみ防止（タップの視覚フィードバックは維持）。常時表示のnoticeは撤去し、カメラ起動失敗時のみカメラ枠内にエラーとリトライボタンを表示、撮影データ生成失敗時のみ画面下部にエラー表示。3枚撮り切って未選択の状態はサムネイル帯のパルス演出とシャッターのグレーアウトのみで伝える。
- `src/components/PhotoPicker.tsx`: 削除。
- `src/components/StoreHome.tsx`: `CameraCapture`から`onBack`プロパティが無くなったことに追従し、渡すのをやめた。

- `src/components/PhotoPreviewOverlay.tsx`: 「これに決める」「この写真を消す」をアイコンのみのボタンに変更。
