# 2026-07-04 枠デザイン作業の記録と反省

この文書は、2026-07-04の枠・ロゴ・管理画面まわりの作業で起きたことを、次スレッドへ正確に引き継ぐための履歴記録です。

現在の最上位原則は `docs/project-principles.md`、現在の基本設計は `dog_sns_design.md` を正とします。この文書は、当時の判断のぶれや反省点を確認するための作業ログとして扱います。

## 結論

このスレッドでは、軽井沢店の枠管理とデザイン調整を進めたが、途中で判断がぶれて、ユーザーが求めていた「正式ロゴ入りで見栄えのよいL版フレーム」から外れた。

次スレッドでは、先に進まず、まず枠デザインを立て直すこと。

## ユーザーの本来の要望

- 社長に見せる可能性があるため、見た目のよい枠にしたい。
- ロゴは単独表示ではなく、枠デザインの中に含めたい。
- ユーザーが渡した正式ロゴ画像を使うべき。
- 写真の上に多少デザインが重なってもよいので、単なる無地枠ではなく、店の雰囲気が出る枠が必要。
- 日付の場所は確保したいが、日付用の箱や余計な背景が邪魔になるのは避けたい。
- 写真はL版横向きの比率に合わせたい。

## 起きた問題

### 1. 正式ロゴを使わずに仮デザインへ進めてしまった

ユーザーはロゴ画像を渡していたが、途中で問題解消を急ぎ、正式ロゴを使わない単純SVG枠へ置き換えた。

これは誤り。ロゴは枠デザインの重要要素であり、社長確認用に見せるなら正式ロゴを使う必要がある。

### 2. 枠を単純化しすぎた

写真との重なりや日付表示の問題を避けるため、単なるクリーム色やグリーンのシンプル枠へ寄せた。

しかしユーザーの目的は「多少写真に重なっても、見栄えのよい枠を見せること」だった。デザイン性を落として問題を消すのは目的と違った。

### 3. デモ枠の自動追加が混乱を増やした

軽井沢店の `store_code` が `DEMO_KARUIZAWA` だったため、ログイン時にテスト枠が何度も復活した。

その結果、管理画面で停止したはずの `標準グリーン`、`季節ピンク`、`イベントゴールド` が再び有効になり、iPadで古い帯や `KARUIZAWA` 文字入り枠が表示された。

### 4. 管理画面の枠UIが分かりにくい

既存の `/admin` の枠タブは、新規追加と既存編集が同時に表示され、右側のプレビューや「枠を保存しました」の表示が紛らわしい。

ユーザーが手作業で正しく整理するには使いにくい。後で大きく作り直すべき。

### 5. 撮影プレビューと完成画像Canvasの整合性が未確認

撮影画面は追加CSSでL版比率へ寄せたが、完成画像生成側 `MosaicCanvas.tsx` には古い帯や枠描画ロジックが残っている可能性がある。

次スレッドでは、撮影プレビュー、編集画面、完成画像、印刷時の4つが同じL版・同じ枠見えになるかを確認すること。

## 現時点の重要な注意

- 現在GitHub上にある `karuizawa-premium-*.svg` は正式ロゴ入りではない。
- これを完成版として扱わないこと。
- ユーザーが渡した正式ロゴ画像を使って、枠画像を作り直す必要がある。
- ただし、プロジェクトの正本はGitHub/Vercel/Supabaseであり、特定PCのローカルフォルダやDropboxを正本にしない。
- ロゴ元画像がローカルにしかない場合は、ユーザーに必要最小限の操作で再提示してもらうか、管理画面からアップロードする流れを整理する。

## 次に避けること

- ロゴを文字だけで代用しない。
- 問題を消すために枠を無地にしない。
- 管理画面で何度も手作業させない。
- iPad確認前に複数店舗・複数色へ広げない。
- `seed.example.sql` やデモ店舗の仮データを本番運用と混ぜない。
- ローカルPCやDropboxを正本にしない。
