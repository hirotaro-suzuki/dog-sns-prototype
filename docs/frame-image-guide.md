# 店舗フレーム画像ガイド（デザイン仕様）

最終更新: 2026-07-13

この文書は、店舗ごとの「枠（フレーム）」画像を新しく作る・依頼するときに必要な技術仕様をまとめたメモである。将来、外部（Codexなど別のAIツールや外部デザイナー）にデザインを依頼する際の元資料として使う。いずれ本格的な引き継ぎドキュメントを作る際は、この内容を土台にする想定。

## 基本仕様

- **サイズ**: 1080 × 1080px の正方形（1:1）。アプリ側（`src/components/MosaicCanvas.tsx`の`drawFixedFrame`）が`context.drawImage(frameImage, 0, 0, 1080, 1080)`で強制的にこのサイズへ引き伸ばして描画するため、正方形・1080角以外だと歪む。
  - 2026-07-13より、Adminの枠アップロードが**正方形でないファイルを拒否**する。PNG/JPEG/WebPは**1080×1080未満も拒否**（SVGは拡大に強いため縦横比のみ確認）。仕様外のファイルはその場でエラーメッセージが出る。
- **形式**: PNG（アルファ透明必須）または SVG。JPEG/WebPもアップロードは通るが透明を扱えないため不可。
- **ファイルサイズ上限**: 2MB（`src/app/api/admin/stores/route.ts`の`MAX_STORE_ASSET_BYTES`）。
- **アップロード先**: Admin管理画面の枠登録機能。Supabase Storageの`store-assets`バケットに保存される。

## 重なり順と「隠す」仕組み

キャンバスへの描画順は「写真 → モザイク → 文字 → 枠画像」で、**枠は必ず一番上に重なる**。さらに写真自体は初期状態でキャンバス全面（1080×1080）を覆うように配置される（ユーザー操作で縮小も可能だが基本は全面表示）。

このため：

- 枠の中央（写真窓）にあたる部分は**必ず透明**にすること。
- 枠として写真の端を「隠したい」場合は、その部分を**線（stroke）ではなく塗り（fill）で不透明にする**必要がある。線だけ（`fill="none"`、`stroke-width`のみ）のデザインだと、線の外側も内側も写真がそのまま見えてしまい、「隠す」効果が出ない。
- 逆に「額縁の外周が不透明な塗り」であれば、その範囲は確実に写真を隠せる（SVGなら`mask`で窓部分だけ透明にする手法が使える）。

## 既存デザインの2つの流儀（参考実測値）

`public/store-frames/`内の実例より。

### A. 極細ボーダー型（現行の葡萄房 `*-square.svg`）

写真がほぼ全面に見え、縁取りは薄い線のみ。**線だけなので写真は隠れない**。

- 外側の線: 端から26px内側、太さ7px
- 内側の線: 端から38px内側、太さ2px
- ロゴ: 写真の上に直接重ねて配置（例: 幅210〜256px程度、下端から190〜230px程度の位置）

### B. 帯（額縁）型（旧デザイン、`karuizawa-simple-cream.svg`等）

上下左右に不透明な帯があり、**写真を実際に隠せる**。

- 上下の帯: 154px（塗りつぶし）
- 左右の帯: 48px（塗りつぶし）
- 二重の縁取り線（写真窓のすぐ内側）: 太さ10px＋3px
- ロゴ: 上の帯の中に配置
- SVGでは`mask`を使い、全体を塗りつぶしてから窓部分だけ透明にする実装例あり

「写真を隠したい」という要望に合わせるなら **B（帯型、塗りつぶし）** をベースにするのが適切。窓の形は正方形に近い比率にすること（過去に窓が横長になり作り直した経緯があるため、`docs/session-2026-07-04-frame-reset.md`参照）。

## 日付表示（アプリが別途重ねて描画）

枠画像の上に、さらにアプリが「YYYY.MM.DD」の日付文字列を描画する。位置・色・サイズは店舗ごとに`store_frames`テーブルの値で決まり、Admin画面の枠登録時に設定できる。

- 座標: `date_x` / `date_y`（0〜1080の範囲、デフォルト 900 / 90）
- フォントサイズ: `date_font_size`（12〜96、デフォルト 38）
- 色: `date_color`（`#RRGGBB`形式、デフォルト`#ffffff`）
- 表示有無: `date_enabled`（デフォルトtrue）

枠デザイン側でこの付近を日付用に空けておくか、空けられない場合はAdmin側で座標をデザインに合わせて変更する。
