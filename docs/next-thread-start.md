現在の大きな方針は以下です。

- 店舗iPadは、店舗コード + PINでログインする
- 店舗情報、担当者一覧、写真枠、表示設定はSupabaseから取得する
- ロゴは単独素材ではなく、写真枠のデザイン内に含める
- 撮影画像は最大3枚まで、長辺2400pxを上限に軽量化してブラウザ内に一時保持する
- 写真選択後に担当者を選択する
- ワンチャン情報はDBに保存せず、写真上に自由配置する短い文字として完成画像に焼き込む
- 印刷は完成画像確認画面の印刷ボタンから行う
- クラウド保存するのは、お客様からSNS掲載OKをもらった完成画像だけにする
- /admin は本部メンテナンス画面として、写真、店舗、担当者、枠を管理する
- SNS投稿、自動投稿、Instagram連携は先走って実装しない

現在の直近テーマは、飲食店オーナーへデモとして見せられるように、本部側のDBメンテナンス画面と店舗側オペレーションを整えることです。
まずは /admin の本部メンテナンス画面から進めています。

/admin で今回すでに反映済みのこと:

- 「かんたん枠登録」パネルを削除済み
- 入口表示を本部管理画面寄りに調整済み

- ログアウト後に再ログインしてもログアウトボタンが出るよう修正済み
- 「再読み込み」ボタンをタブ内に見える位置へ移動済み

/admin で次に進む候補:

- 写真一覧と詳細を分ける
- 一覧で店舗順、日付順、状態順などに対応する
- 複数写真を選択できるようにする
- 選択した写真に一言メモを順に追加できるようにする
- 選択した写真を非表示、投稿候補、保留、使用しないなどに整理できるようにする

次のチェックポイントは、写真タブのDB/API土台です。

- `assets.short_caption` は本部が追加する一言メモとして使う
- 一言メモは短文前提で、まず120文字程度を上限にする

- 既存写真は初期状態 `new` でよい
- SQLはmigrationとしてGitHubへ残し、実際のSupabase SQL Editorへの適用はユーザーが行う
- API一覧取得で一言メモと確認状態を返す
- API個別更新で一言メモと確認状態を更新できる
- 不正な状態値はAPIで拒否する
