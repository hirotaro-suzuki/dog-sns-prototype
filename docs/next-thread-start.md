# 次のCodexスレッド開始時の指示文

次のCodexスレッドで開発を再開するときは、最初に以下をそのまま貼る。

```text
GitHub の hirotaro-suzuki/dog-sns-prototype の main が最新版です。
GitHub main を正として作業してください。

まず GitHub 上の README.md と dog_sns_design.md を必ず読んでください。
dog_sns_design.md が、現時点の設計・開発方針の正本です。
必要に応じて docs/supabase-handoff.md、docs/ownership-handoff.md、docs/session-2026-06-25.md も読んでください。

このプロジェクトは、GitHub、Vercel、Supabase を正として進めます。
ローカルPC上の作業環境や複製リポジトリを前提にしないでください。
クラウド中心にしている最大の理由は、次のCodexスレッドや別作業者へ引き継ぎやすくするためです。
引き継ぎに手間のかかる構成や、特定のPC・特定フォルダ・特定個人の手元環境に依存する開発は避けてください。

開発判断はCodexを中心に進めます。
以前のように、ターミナル画面を別AIへ渡して判断を仰ぐ運用は前提にしません。
将来SNS投稿文の自動生成などでAI機能が必要になった場合は、その時点で用途と設計を整理してから検討します。

現在の大きな方針は以下です。

- 店舗iPadは、店舗コード + PINでログインする
- 店舗情報、担当者一覧、ロゴ、写真枠、表示設定はSupabaseから取得する
- 撮影画像は最大3枚まで、長辺2400pxを上限に軽量化してブラウザ内に一時保持する
- 写真選択後に担当者を選択する
- ワンチャン情報はDBに保存しない
- ワンチャン情報は、写真上に自由配置する短い文字として完成画像に焼き込む
- モザイクと文字は写真に連動して動く
- 印刷は完成画像確認画面の印刷ボタンから行う
- クラウド保存するのは、お客様からSNS掲載OKをもらった完成画像だけにする
- 撮影途中の写真、選ばれなかった写真、未許諾写真、キャンセル写真はクラウドへ送らない
- 完成画像確認画面の保存ボタンから、Supabase Storageとassetsテーブルへ保存できる
- `/admin` で保存済み写真の一覧、説明文更新、非表示/復帰ができる
- `/admin` で店舗マスタと担当者マスタを編集できる
- seed.example.sql は本番運用では使わず、新規環境の動作確認用サンプルとして残す

2026-07-04時点の補足:
- Vercelの `ADMIN_MAINTENANCE_PIN` は再設定済み。PINの値はGitHubには記録しない。
- デモ店舗 `TOKYO / 0000` では、テスト枠3種類が店舗画面に表示されることをiPadで確認済み。

次に進む候補は、保存済み写真からSNS投稿内容を作成・保存する機能です。
その前に、Vercelの最新デプロイ状態、`/admin` の店舗・担当者マスタ編集、店舗側の完成画像保存が実機で期待どおり動くか確認してください。

私は73歳の素人で、バイブコーディングにチャレンジ中です。
できる限り、SQL確認、GitHub確認、Vercel確認、コード修正、手順整理などはCodex側で進めてください。
私が画面操作しないと進められない場合だけ、専門用語を減らして一歩ずつ案内してください。

各セッションの終わりや区切りでは、作業内容の記録に加えて、気づいた点や次に良くなる提案を1つ以上ください。
```
