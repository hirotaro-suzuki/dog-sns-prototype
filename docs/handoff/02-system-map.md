# しくみの図解（システムはどんな部品でできているか）

最終更新: 2026-07-20

このシステムは、4つのサービス（インターネット上の借り物の場所）を組み合わせてできています。
お店にたとえると、こうなります。

```
        お店のiPad（道具）
            │
            │ インターネット
            ▼
   ┌────────────────────┐
   │  Vercel（バーセル）        │  ← お店の「店先」。画面を表示する場所
   │  お客様が触れる画面すべて    │
   └────────┬───────────┘
            │
            ▼
   ┌────────────────────┐
   │  Supabase（スパベース）    │  ← お店の「金庫」。大事なものを全部しまう
   │  ・保存した完成写真          │
   │  ・店舗の名簿、担当者の名簿   │
   │  ・写真枠のデザイン          │
   └────────────────────┘

   ┌────────────────────┐
   │  GitHub（ギットハブ）      │  ← 「設計図の保管庫」。画面や仕組みの
   │  プログラムと説明書の正本     │     作り方が全部ここに残っている
   └────────────────────┘
      （Vercelは、この設計図を見て店先を自動で建て直す）
```

## 4つのサービスの役割と入り口

| サービス | たとえ | 中にあるもの | 誰が触るか |
|----------|--------|--------------|------------|
| Vercel | 店先 | 店舗画面と本部画面の公開場所（https://dog-sns-prototype.vercel.app） | 全員が画面として使う。設定はオーナーと技術担当だけ |
| Supabase | 金庫 | 完成写真、店舗・担当者の名簿、写真枠、PINなどの設定 | 技術担当だけ。日常のデータ管理は本部画面から行う |
| GitHub | 設計図の保管庫 | プログラム本体と、すべての説明書（docs/フォルダ） | 技術担当だけ |
| iPad | 道具 | 何も保存されない。画面を映すだけ | 店舗担当者 |

## 覚えておくと安心なこと

- **iPadには写真は残りません。** 保存した写真はすべて金庫（Supabase）にあります。iPadを落としても壊しても、保存済みの写真は無事です。
- **設計図（GitHub）を書き換えると、店先（Vercel）が自動で建て直されます。** これは便利ですが、裏を返すと、設計図をうかつに触ると本番の画面がすぐ変わってしまう、ということです。設計図を触るのは技術担当だけにしてください。
- **金庫（Supabase）を直接開ける必要は、ふだんはありません。** 店舗・担当者・枠・写真の管理は、ぜんぶ本部画面 /admin からできます。金庫を直接触るのは、修理や特別な整理のときだけで、技術担当の仕事です。
- 4つのサービスには、それぞれ持ち主のアカウント（会員登録）があります。誰が持ち主になるかは、資料6（権利・アカウント引き継ぎ手順書）で説明します。

## ログイン方法のまとめ

| 誰が | どこに | 何を使って |
|------|--------|------------|
| 店舗担当者 | 店舗画面 | 店舗コードとPIN（お店ごとに違う） |
| 本部担当者 | /admin | 本部PIN（1つだけ。大切に管理する） |
| 技術担当 | GitHub・Vercel・Supabase | それぞれのアカウント（資料6・7参照） |
