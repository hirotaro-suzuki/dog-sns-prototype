# プロジェクト原則

最終更新: 2026-07-07

この文書は、店舗分散型SNS自動投稿システムを進めるうえで、スレッドが変わってもぶれないための最上位方針です。

細かな画面仕様やDB仕様は `dog_sns_design.md` を正とします。ただし、判断に迷った場合は、この文書の原則を優先します。

新しいCodexスレッドの開始手順は `docs/START_HERE.md`、読む文書の分類は `docs/DOC_INDEX.md` を参照します。

## 1. 引き渡せる形を最優先する

このプロジェクトは、ユーザー本人の長期事業ではなく、友人が経営する飲食店を支援するためのものです。

ある程度使える形になったら、店舗側または運用主体へ引き渡し、ユーザー本人は手を引く前提で進めます。

そのため、短期的に作りやすいことよりも、次の人が理解し、引き継ぎ、運用できることを優先します。

## 2. ユーザー本人のPCや記憶に依存しない

ユーザー本人が、将来すべての経緯、判断、設定、操作手順を覚えている前提にしません。

特定のPC、Dropbox、個人フォルダ、ローカル複製リポジトリがないと再開できない状態にしません。

仕様、判断理由、作業手順、未決事項、注意点はGitHub上の文書に残します。

## 3. クラウドとCodespacesを正とする

このプロジェクトでは、GitHub、Codespaces、Vercel、Supabaseを正とします。

- ソースコードと文書は GitHub `main`
- 実装、ビルド、コミット、pushはユーザーがGitHub Codespacesターミナルで行う
- 公開、デプロイ、ビルド結果の確認は Vercel
- DB、Storage、Auth、店舗設定、担当者、枠、完成画像は Supabase
- ローカルPC、Dropbox、手元コピーは正本にしない

Codexは、備え付けのGitHubプラグインを使った直接ファイル編集やpushを行いません。変更が必要な場合は、ユーザーがCodespacesターミナルへコピペして実行できる短いコマンドまたはスクリプトを提示します。Codespacesターミナルで進められない場合は、ローカルやDropboxへ逃げず、何ができて何ができないかをユーザーへ説明して止まります。

GitHubへ反映したこと、Codespacesでビルドが通ったこと、Vercelのデプロイが成功したこと、Vercel上で実操作できたこと、SupabaseへSQLや設定を適用したことは、それぞれ別物として扱います。

## 4. Codexは先走らない

目的、OK条件、保存先、運用者、影響範囲が曖昧な機能は、実装前に候補を出してユーザーと確認します。

実装前に、少なくとも次を整理します。

- 今回の目的
- 何ができたらOKか
- 確認方法
- 影響する画面、DB、Storage、外部サービス
- 更新すべき文書

ここでいうTDDは、自動テストだけでなく、実装前に「何ができたらOKか」を決める姿勢を含みます。

## 5. 現場操作を軽くする

店舗側の主な利用者は、忙しい飲食店の現場スタッフです。

店舗側iPad画面は、説明を読まなくても迷いにくく、撮影、編集、印刷、同意後保存までを短い手順で進められることを重視します。

高度な管理、設定、確認作業は本部側 `/admin` に寄せ、店舗側へ複雑さを持ち込みません。

## 6. 店舗側、本部側、SNS連携を疎結合にする

このプロジェクトは、まず以下の流れを安定させます。

1. 店舗側で撮影、編集、印刷する
2. SNS掲載OKの完成画像だけをSupabaseへ保存する
3. 本部側で保存済み画像を確認、整理する

SNS投稿、自動投稿、投稿本文生成、Instagram連携は後続フェーズです。必要性、権限、運用者、失敗時対応が整理されるまで先走って実装しません。

## 7. 文書は増やしすぎない

文書は、次の人が迷わず再開するために残します。ただし、文書が増えすぎると次スレッドの理解を妨げます。

新しい文書を作る前に、既存文書へ最小限追記できないか確認します。

作業ログは必要に応じて残しますが、毎回読む前提にしません。どの文書を読むべきかは `docs/DOC_INDEX.md` に整理します。

## 8. 報告は確認済みと未確認を分ける

GitHubに反映したこと、Codespacesでビルドが通ったこと、Vercelでデプロイが成功したこと、Vercel上で実操作確認したこと、iPad Safariで確認したこと、SupabaseにSQLや設定を適用したこと、実機印刷したことは別物として扱います。

作業報告では、確認済みと未確認を分けて伝えます。

未確認のものを確認済みのように書きません。
