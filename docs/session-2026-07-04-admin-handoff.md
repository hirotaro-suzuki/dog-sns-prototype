# 2026-07-04 管理画面ブラッシュアップ引き継ぎ

最終更新: 2026-07-04

この文書は、`/admin` 本部メンテナンス画面のブラッシュアップ作業を次スレッドへ安全に引き継ぐための作業ログである。

GitHub `main` を正本とする。ローカルPC、Dropbox、手元フォルダ、複製リポジトリを正本にしない。

## この作業の目的

飲食店オーナーへデモとして見せられるように、本部側の管理画面を「テスト画面」ではなく、ちゃんとデザインされ、操作が簡単に見える業務画面へ整える。

当面はSNS投稿機能そのものへ進まず、保存済み写真を本部で確認、整理、候補化できる状態を作る。

## 合意した管理画面方針

- `/admin` は本部メンテナンス画面として扱う。
- 中に入った後のタブは、写真、店舗、担当者、枠を基本にする。
- 写真一覧と詳細は分ける。
- 写真は毎日4店舗から各10枚程度、合計40-50枚上がる前提で扱う。
- 初期表示は「今日」の写真を標準にする。
- 複数日を表示する場合は、店順と日付順で見られるようにする。
- 店順は `stores.sort_order`、同順の場合は店舗表示名を優先する。
- 日付順は新しい順を標準にし、古い順にも切り替えられるようにする。
- 日付表示の後ろに不要な括弧表示を出さない。
- 開始日、終了日の入力欄も、iPad Safariの曜日括弧表示を避けるため `YYYY-MM-DD` の通常入力として見せる。
- 店舗側で画像に焼き込んだ文字の再編集はしない。
- 本部側の写真メモは40文字以内の一言メモに一本化し、旧来の説明文入力は画面に出さない。
- 本部側では、サムネイル一覧で複数写真をチェック選択し、詳細画面で一言メモや確認状態を順に処理できるようにする。
- 将来的に、選択した写真を投稿候補にして投稿へ進む流れはあり得るが、今は実装しない。

## GitHub mainへ反映済みのこと

- `/admin` に残っていた「かんたん枠登録」パネルを削除した。
- `/admin` の入口表示を、本部管理画面として見えるように調整した。
- 管理画面にログアウト導線を追加した。
- ログアウト後に再ログインしてもログアウトボタンが出るように修正した。
- `assets.short_caption` と `assets.review_status` の追加migrationをGitHubへ追加した。
- `supabase/schema.sql` に `short_caption`、`review_status`、制約、検索用indexを反映した。
- Supabase型定義へ `short_caption` と `review_status` を反映した。
- `/api/admin/assets` の一覧取得で一言メモと確認状態を返すようにした。
- `/api/admin/assets` の一覧取得で `sortMode=store|date`、`dateOrder=desc|asc`、`reviewStatus` を受けられるようにした。
- `/api/admin/assets/[id]` の個別更新で一言メモと確認状態を更新できるようにした。
- 一言メモは40文字以内、不正な確認状態はAPIで拒否するようにした。
- 写真タブUIで、確認状態、一言メモ、確認状態フィルター、日付順/店順、日付の新しい順/古い順を操作できるようにした。
- 写真カード上に確認状態と一言メモを表示するようにした。
- 写真タブを、サムネイル一覧画面と写真詳細画面に分けた。
- 写真一覧でチェックボックスによる複数選択ができるようにした。
- 「表示中をすべて選択」「選択解除」「選択した写真を確認」を追加した。
- 複数選択した写真を、詳細画面で `前へ`、`次へ`、`保存して次へ` により順番に確認できるようにした。
- 写真を単体で押した場合も、詳細画面へ進めるようにした。
- 詳細画面から「一覧へ戻る」で一覧へ戻れるようにした。
- 写真詳細画面から旧来の `説明文` 入力欄と `説明文を保存` ボタンを外した。
- 写真一覧と詳細の撮影日時表示を、括弧なしの短い形式にした。
- 開始日、終了日の入力欄は、iPad Safariの標準日付入力による曜日括弧表示を避けるため、標準日付入力を隠し、同じ値を同期する `YYYY-MM-DD` の代理入力欄を表示する `AdminDateInputNormalizer` を追加した。

## 注意が必要な既存実装

日付の括弧表示には2種類ある。

- 写真カードや詳細の撮影日時表示は、最新コードでは `月/日 時:分` の形式で表示する。
- 開始日、終了日の入力欄は、iPad Safariの標準日付入力が曜日を括弧つきで表示することがあるため、`AdminDateInputNormalizer` で標準日付入力を隠し、同じ値を同期する `YYYY-MM-DD` の代理入力欄を表示する。

店舗ロゴ管理は、今後削除または非表示にする予定の古い機能としてまだ一部残っている。今回の写真タブ改善とは別作業にする。

## 実装前に確認済みのOK条件

- `assets.short_caption` は一言メモとして使う。
- 一言メモは短文前提で、40文字以内にする。
- `assets.review_status` の内部値は `new`、`candidate`、`hold`、`rejected` とする。
- 表示名は `未確認`、`投稿候補`、`保留`、`使用しない` とする。
- 既存写真は初期状態 `new` でよい。
- APIは一覧取得で一言メモと確認状態を返し、個別更新で一言メモと確認状態を更新できるようにする。
- 不正な状態値はAPIで拒否する。
- 初期表示は今日の写真でよい。
- 店順は `stores.sort_order` 優先でよい。
- 日付順は新しい順を標準とし、古い順にも切り替えられるようにする。
- 写真一覧と詳細は同時表示ではなく、別画面として見せる。
- 一覧では複数写真をチェック選択できるようにする。
- 詳細では選択した写真を順番に確認し、一言メモと確認状態を保存できるようにする。
- 旧来の説明文入力は画面から外し、一言メモに一本化する。
- 開始日、終了日の入力欄は、曜日括弧が出ない表示にする。
