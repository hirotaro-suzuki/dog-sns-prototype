## 3. 追加修正: 編集画面のキャンバスが縦長になる不具合（同日、実機テストで発見）

iPad実機確認で「編集画面だけ画像が若干縦長になる」との報告。原因は `.canvas-frame` の
「height:100% + aspect-ratio」方式で、残り領域の高さが横幅を超えると、幅だけ
`max-width:100%` で押さえられて高さが100%のまま残り、箱が縦長になるため
（CSSでは高さの指定が縦横比より優先される）。撮影画面は下部の部品が多く
「残り高さ＜横幅」のため発症せず、完成画面は `object-fit: contain` のため無事だった。

対応: `MosaicCanvas` がResizeObserverで残り領域を測り、幅・高さの小さい方を
`--canvas-frame-side` としてCSSへ渡し、枠の一辺をそれに固定する方式へ変更。
タッチ座標は枠の実寸から毎回計算しているため、位置ずれは起きない。
なお撮影画面の `.camera-stage` にも同じ理屈のリスクは残るが、映像が
`object-fit: cover` のため実害が出ておらず、今回は触っていない。

## 4. 追加改善: 文字入力を白い固定パネルへ分離（同日、実機テストの指摘）

従来は、写真上に描かれた文字の真上へ半透明の入力欄を重ねていたため、同じ文字が
二重に見え、どこで入力するのかも分かりにくかった。ユーザーと合意のうえ、次へ変更:

- 写真上の透明な入力欄を廃止。写真上の文字は描画された1つだけになり、ダブりが消えた。
  選択中の目印は従来どおり点線の囲み。
- 文字を選ぶと、キャンバス**上端**に白いパネルを重ねて表示。中に「白地に黒文字の
  入力欄（プレースホルダー: ここに文字を入力）」と、従来のサイズ・色・削除・閉じる・
  残り文字数をまとめた。上端にした理由は、iPadのキーボード（画面下半分）に隠れないため。
- 入力欄の文字色は常に黒（白地で見やすく）。選んだ色は写真上の文字で確認する。
- 文字の移動は従来どおり写真上の文字を直接ドラッグ（キャンバス側の既存実装で動作、
  浮遊入力欄専用だったドラッグ処理・位置計算コードは削除）。
- パネル外（キャンバス）タップで閉じる動きは従来どおり。

最終形その3（同日）: その2の後、「キーボードが出ると写真下端のパネルごと隠れる」と実機報告。
原因は縮小処理の発動条件（見える高さが80px以上減＋ズーム倍率がぴったり1）がiPadOS 26実機で
すれ違い、縮みが発動しないまま押し上げ打ち消しだけが効いていたため。対応として条件発動を
やめ、編集画面の表示中は常に visualViewport の高さを `--app-viewport-height` へ反映し続ける
方式に変更（キーボードが出れば見える高さは必ず減るので判定のすれ違いが起きない。
ピンチズーム中のみ除外）。

最終形その2（同日）: 「横一列パネルではチェックがアイコン列の下の行のままになる」との指摘で、
ユーザー確認のうえ次の構成に変更。①下段はアイコン5個＋チェック（右端寄せ）の1行だけに統合。
チェックはアイコンと同じ高さになり、下段の高さは常に固定。②文字パネルは「入力欄＋
ボタン1列」の縦2段箱（幅最大390px）とし、**写真の下端に重ねて**表示（枠の柄と重なるのは
ユーザー了承済み。`.canvas-frame` の内側はuser-select:noneで入力不可のため、wrap直下に
絶対配置）。パネル開閉で写真サイズが変わらなくなった。③iPadOS 26.5.2でも押し上げ
スクロールが残ると実機報告があったため（`interactive-widget` は出力確認済み）、編集画面
表示中はwindow/visualViewportのscrollを監視して即座に打ち消す方式へ強化。

最終形（同日）: 実機で「キーボードで押し上げスクロールが残る」「パネルが大きすぎて
チェック（確定）ボタンが一段下がる」と報告があり、ユーザー判断で次を実施。
①文字色を白・黒の2色に削減（どちらも縁取り付きで描画されるため実用上十分。
5色のままでは横一列が縦持ち幅に収まらない計算だった）。②パネルを「入力欄＋全ボタン横一列」
の1行構成にし、チェックボタンの行（76px）内に収めた。パネル表示による縦の消費はゼロになり、
チェックの高さも従来どおり。入力欄は flex で15文字相当（最大320px）まで、狭い機種では
入力欄が先に縮む。③viewportに `interactiveWidget: "resizes-content"` を追加
（iPadOS 18.4以降でキーボード時にページを押し上げず表示領域を縮める公式設定。
古いOSでは無視され、既存のvisualViewport縮小が保険として残る）。
なお「パネルを枠の柄に重ねる」案も検討したが、1行パネルは既存のチェック行に収まり
縦の消費ゼロのため、重ね配置の利点が無く不採用（将来縦を稼ぐ際の選択肢としてメモ）。

再追記（同日）: キャンバス上に置く案は「キーボードで写真の下半分が隠れる」と実機報告があり、
ユーザー提案で最終形へ変更。①パネルは下段のチェック（確定）ボタンの右へ移動し、
「入力欄1段＋ボタン1列（水平・サイズ3・色5・削除・閉じる）」の薄型に。「あとXX文字」表示は
廃止（15文字上限は入力欄のmaxLengthが引き続き強制）。②キーボード表示中は、
visualViewportで見えている高さを実測して `--app-viewport-height` に入れ、
`.store-app-shell` 全体をその高さへ縮める。写真は小さい正方形になって全体が見え、
キーボードが引っ込めば元に戻る（ピンチズーム中は反応しないようガード済み）。

追記（同日）: 上端重ねの初版では「入力欄にカーソルが立たず入力できない」と実機報告。
原因は `.canvas-frame` の `user-select: none`（写真操作用の選択禁止設定）で、その内側に
置いた入力欄はiPad Safariでフォーカスできなかった。パネルを**キャンバスの外**
（上部ツールバーとキャンバスの間、通常フロー）へ移動して解決。パネルが出た分は
`--canvas-frame-side` の実測がキャンバスを自動で縮めるため、正方形とスクロール無しは
維持される。
