# AGENTS.md

## 作業規則

- 正式な作業場所は `/workspaces/dog-sns-prototype` とし、GitHub の `main` を基準にする。
- ローカルPC、Dropbox、手元フォルダ、古い複製リポジトリは使用しない。
- GitHub連携は読み取り専用とし、書き込みは Codespaces 内の `git` を使う。
- 作業前に `git status` を確認し、既存の変更や未追跡ファイルを勝手に削除・上書きしない。
- `docs/DOC_INDEX.md` を入口に、作業に必要な文書だけを読む。
- 大きな変更は、目的・範囲・確認方法を事前に説明してから着手する。
- 修正は小さく行い、変更後に適切な方法で検証する。
- APIキー、パスワード、トークンなどの秘密情報を Git に保存しない。
- 作業の節目で `/status` を確認し、進行状況と残作業を把握する。
- Codex自身が実行できる編集、確認、コマンド操作を、利用者へコピー＆ペースト依頼しない。

## 利用者への説明と進め方

- 利用者は開発の初心者であることを前提に、専門用語をできるだけ避け、必要な場合は簡単な言葉で意味を説明する。
- 一度に多くの内容を詰め込まず、今から何をするか、何が終わったか、次に何をするかを順番に伝える。
- 利用者の明確な依頼がない限り、調査から編集、DB操作、commit、pushなどへ勝手に進まない。
- 重要な変更や元に戻しにくい操作は、目的と影響を簡単に説明し、利用者の確認を得てから実行する。
- 利用者が判断しやすいように選択肢とおすすめを示し、返答を待ってから次の段階へ進む。
